Using Parameters

There is support for parameterised statements, and using parameters when calling procedures.

Input parameters are added to a Request using Request.addParameter(), and output parameters are added using Request.addOutputParameter()

Parameter names

In T-SQL, parameter names are identified by being prefixed with an '@'.

{% highlight sql %} select id from address where first_name = @name and age > @age {% endhighlight %}

The parameters names used in this API do not include the '@'.

{% highlight javascript %} request.addParameter(..., 'name', ...); request.addParameter(..., 'age', ...); {% endhighlight %}

Input Parameters

This is how input parameters might be used when executing a statement.

{% highlight javascript %} var TYPES = require('tedious').TYPES; var sql = 'select id from address where first_name = @name and age > @age'; var request = new Request(sql, function(err) { .... }); request.addParameter(TYPES.VarChar, 'name', 'Fred'); request.addParameter(TYPES.Int, 'age', 42); connection.execSql(request); {% endhighlight %}

Calling a stored procedure with parameters works in a similar way.

{% highlight javascript %} var TYPES = require('tedious').TYPES; var request = new Request('get_addresses', function(err) { .... }); request.addParameter(TYPES.VarChar, 'name', 'Fred'); request.addParameter(TYPES.Int, 'age', 42); connection.callProcedure(request); {% endhighlight %}

Output Parameters

The values for output parameters are provided with emitted returnValue events.

{% highlight javascript %} var TYPES = require('tedious').TYPES; var request = new Request("select @number=42, @string='qaz'", function(err) { .... }); request.addOutputParameter(TYPES.Int, 'number'); request.addOutputParameter(TYPES.VarChar, 'string'); request.on('returnValue', function(parameterName, value, metadata) { console.log(parameterName + ' = ' + value); // number = 42 // string = qaz }); connection.execSql(request); {% endhighlight %}
